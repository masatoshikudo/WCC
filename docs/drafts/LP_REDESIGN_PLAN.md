# LP UI 全面見直し プラン v1（叩き台）

> **位置付け**: LP P3 後続の UI レベル全面改訂のための判断ドキュメント
> **起点**: `docs/drafts/LP_CURRENT_STATE.md`（棚卸し）+ Social Sisters 観察（PC / SP）
> **基本姿勢**: ビジュアル原則は Social Sisters に揃える / コピーの世界観は WCC 独自軸を維持する
> **対象ブランチ想定**: `feat/lp-redesign-v1`
> **ステータス**: 起草中（5 論点判断待ち）
> **作成日**: 2026-05-06
> **作成者**: Claude Desktop（設計役）

---

## 1. 背景と目的

### 1.1 経緯

LP P3 で確立した「ベンチマーク = Social Sisters」「ヒーローコピー = 翌朝軸」「禁句 = 感動・永遠」の方針は、**ブランドコピーレベルの確定** までで止まっていた。UI レベル（情報設計・タイポグラフィ・レイアウト・余白のリズム）への翻訳は未着手で、現状 LP は LP P2 までの構造を引き継いでいる。

LP P3 残債修正（A 群 4 件）が完了し、現状 LP には明確なバグ・情報不整合がない状態になった。ここで UI レベルの全面見直しに踏み込む。

### 1.2 目的

Social Sisters の UI 原則を WCC LP に翻訳し、以下を達成する:

- 視覚的洗練度を Social Sisters 相当まで引き上げる
- 情報設計を刈り込み、ファーストビューでブランドメッセージに集中させる
- 9:16 縦動画を「サービスの output」として最大限活かす配置に再設計する
- WCC 独自のコピー世界観（翌朝・ふたり・余韻）はそのまま維持する

### 1.3 非対象

以下は本プランの対象外:

- ブランドコピーの再策定（LP P3 で確定済み、本プランでは維持）
- 禁句リストの改訂（永遠・感動は引き続き禁句）
- `/book` ページ（予約フロー）の構造変更
- `/admin` ページ（運営用画面）
- `/legal/*` ページ

## 2. 観察まとめ（Social Sisters）

### 2.1 観察済みページ（PC / SP 両方）

- `/wedding-social-media`（Weddings = LP トップ相当）
- `/services`（Pricing）
- `/contact`

### 2.2 PC で観察された UI 原則（10 項目）

1. **ヒーローはテキストのみ**。動画・写真なし。見出し + サブコピー + 2 ボタン
2. **背景は単色ベージュ**（薄い暖色、`#EFEBE5` 程度）で全セクション統一
3. **テキストは黒**（純黒に近い）
4. **CTA は黒地白文字の完全な楕円ピル**、アクセント色なし
5. **見出しはディスプレイセリフ**（Tan Mon Cheri 系、weight 細め）
6. **本文・UI ラベルはサンセリフ**、UI ラベルは全大文字 + 字間広め
7. **9:16 縦動画は 3 列グリッド固定**（マルキー的な動きなし）
8. **動画にキャプション**（couple 名 + venue + 1 段落）
9. **「テキストが動画に乗る」設計はゼロ**。テキストとビジュアルは常に隣接
10. **セクション間余白が極めて大きい**（縦スクロールの「呼吸感」）

### 2.3 SP で追加観察された原則（5 項目）

11. **9:16 動画は SP では縦 1 列**、6 本連続でしっかり観てもらう設計
12. **ヘッダーは「ロゴ + ハンバーガー」のみ**、SNS / CTA はメニュー内
13. **ヒーローのテキストサイズが大きい**（30〜40px 相当）
14. **フッター写真モザイクが SP では 3 列正方形グリッド**で締まる
15. **ボタン形状は PC と完全一致**（楕円ピル、サイズだけ若干調整）

### 2.4 WCC との根本的な差分

| 観点 | Social Sisters | WCC 現状 |
|---|---|---|
| 実績 | 250 件（3 年） | 0 件（M1 進行中） |
| ヒーローでの説得 | テキストのみ（実績で信頼担保） | 動画 + テキスト（output で信頼補完） |
| セクション数 | 5 程度 | 7 |
| 背景カラー切り替え | なし（単色） | あり（白 / 黒 / ベージュ） |
| 動画の見せ方 | 固定グリッド（観てもらう） | マルキー（ちらつかせる） |
| FAQ | Pricing ページに分離 | LP に内蔵 |

storytelling through an iPhone

We will be on hand to capture all your special moments...

[Pricing]  [Contact]

```

WCC 翻訳:

```

#### 案 B: ヒーロー左右分割

- WCC 独自軸として「実績ゼロでも 1 本だけ作例を見せる」中間解

#### 案 C: 案 A + 直後第 2 セクションを動画グリッド最上位化（**γ 案、Masatoshi 確定済み**）

- 実績ゼロをカバーする「output 即提示」を、ヒーローではなく第 2 セクションで担保

#### 私の推奨: **案 C（γ 案）**

### 論点 2: カラーパレット再設計

**現状**: 白 / 黒 / ベージュの切り替え + アクセント `#ca5311`（オレンジ系） + アクセント hover `#b84a0f`。

**目標**: 単色ベージュ背景 + 黒テキスト + 黒ピルボタン + アクセント色廃止。

#### 案 A: Social Sisters 完全模倣

- ベース背景: 単一ベージュ（`#EFEBE5` 程度の暖色グレージュ）
- セクション切り替えなし（全セクション同色）
- テキスト: 純黒 `#000000`
- CTA: 黒背景 + 白文字
- リンク: 黒（下線）
- アクセント色: 完全廃止
- 区切り線: 細いグレー（`#D5D2C8` 程度）

#### 案 B: ベース 2 色（メイン + サブ）

- メイン: 単一ベージュ
- 一部セクション（例: フッター CTA セクション）のみ別色（例: より濃いベージュ）
- 視覚的リズムを保つ

#### 案 C: ベース 3 色（メイン + サブ + ダーク）

- メイン: ベージュ
- サブ: 白に近い極淡ベージュ
- ダーク: 黒（フッターのみ）
- 既存の WCC 構造に近い、最も保守的な変更

#### 私の推奨: **案 A**

「全面的に揃える」方針なので、最も忠実な案 A を選ぶのが筋。案 B / C は中途半端で、結果として「Social Sisters でも WCC 現状でもない」中間状態になりがち。

- アクセント色廃止の影響範囲（CTA / リンク / ホバー / フォーカス）を一括で黒系に統一する確認

### 論点 3: タイポグラフィ再設計

**現状**: 3 フォント体制（Plus Jakarta Sans / Noto Sans JP / Cormorant Garamond）、見出しは Plus Jakarta Sans 主体、ロゴと一部 H2 のみ Cormorant。

**目標**: ディスプレイセリフ + サンセリフの 2 フォント体制 + 全大文字 UI ラベル + 字間広め。

#### 案 A: Social Sisters 完全模倣（フォント変更込み）

- ロゴ・見出し: **Tan Mon Cheri** または **Editorial New** 系のディスプレイセリフ（要新規導入）
- 本文: Noto Sans JP 維持
- UI ラベル: Plus Jakarta Sans 維持（全大文字 + 字間 0.1em）
- → ロゴフォント方針 α-1（前セッション議論）相当。**Masatoshi はこれを採用しない判断**。

#### 案 B: Cormorant のまま攻める（**α-2 案、Masatoshi 確定済み**）

- ロゴ・見出し: **Cormorant Garamond 維持**、ただし以下を攻める:
  - サイズを大きく（ヒーロー H1 は 60〜80px）
  - weight を細く（300 〜 400、現状 500-700 から軽くする）
  - 字間を広く（0.02〜0.04em）
  - 大文字小文字の使い分けを意識
- 本文: Noto Sans JP 維持
- UI ラベル: Plus Jakarta Sans 維持（全大文字化を強化）
- 結果: 「Cormorant で Social Sisters のディスプレイ感を再現する」

#### 案 C: Plus Jakarta Sans 廃止 + 2 フォント体制

- 見出し: Cormorant Garamond
- 本文 + UI: Noto Sans JP のみ（Plus Jakarta Sans 廃止）
- UI ラベルは Noto Sans JP の太字 + 全大文字 + 字間広め
- フォント数を 2 に絞れる

#### 私の推奨: **案 B**（Masatoshi 確定済み）

理由: ロゴフォント変更はブランド識別子の変更を意味し、admin / book / メールテンプレート等への波及リスクがある。Cormorant のまま組版を攻めれば、Social Sisters の視覚的方向性は十分再現できる。

#### Masatoshi 判断項目（案 B 内の詳細）

- (1) ヒーロー H1 のサイズ感（推奨: `clamp(2.75rem, 6vw, 5rem)` = 44〜80px。現状の `clamp(2.35rem, 5vw, 4.25rem)` より大きく）
- (2) ヒーロー H1 の weight（推奨: **300**、軽さで攻める。現状は 500〜700）
- (3) ヒーロー H1 の字間（推奨: `0.02em`、現状の `-0.035em` から逆方向に攻める）
- (4) UI ラベル全大文字化のスコープ:
  - ナビ「価格 / ギャラリー」→ 「PRICING / GALLERY」（**英語化**）
  - CTA「まず相談する」→ そのまま日本語（全大文字化不可）
  - フッターラベル「USEFUL LINKS / USEFUL BLOGS」→ 既に英語、全大文字化済み（Social Sisters と同じ）
  - 推奨: **ナビは英語化 + 全大文字、CTA は日本語のまま**
- (5) 全大文字 UI ラベルの字間（推奨: `0.1em` 〜 `0.15em`、Social Sisters 相当）

### 論点 4: 情報構造の刈り込み

**現状**: 7 セクション（hero / service-detail / highlights / what-is-wcc / pricing / service-flow / faq）+ FooterCtaSection。

**目標**: 5 セクション程度に削減 + FAQ / service-flow を Pricing ページに分離。

#### 案 A: Social Sisters 完全模倣

LP トップ:
1. ヒーロー（テキストのみ）
2. Highlights（9:16 動画グリッド）
3. What is（左見出し + 右本文 2 カラム）
4. Do I need（左見出し + 右本文 2 カラム）
5. Capturing memories CTA + フッター

別ページ `/pricing`:
- パッケージ詳細
- Bespoke
- Next Steps（service-flow 相当）
- FAQ

別ページ `/book` または `/contact`:
- 予約フォーム

#### 案 B: 段階的削減

LP トップ:
1. ヒーロー（テキストのみ）
2. Highlights（動画グリッド）
3. service-detail（What is 相当に再編）
4. pricing（既存維持）
5. service-flow（簡略化、3 ステップ程度に圧縮）
6. faq（既存維持）
7. フッター CTA

→ M1 では情報量を保つ、M2 以降に段階的に分離。

#### 案 C: 中間解

LP トップ:
1. ヒーロー
2. Highlights
3. What is（service-detail 改）
4. pricing
5. フッター CTA

別ページ `/pricing` 詳細:
- service-flow
- faq
- 既存パッケージ詳細

→ pricing は LP に主要情報を残し、詳細は別ページへ。

#### 私の推奨: **案 C**

理由は 3 つ:

第一に、案 A は「FAQ を Pricing に分離 + service-flow を Pricing に統合」を意味するが、**WCC の予約フローでは FAQ と service-flow が「予約前の信頼担保」として機能している**。Social Sisters は実績 250 件で信頼が担保されているため LP にこれらを置く必要がないが、WCC は事業フェーズが違う。

第二に、案 B は「Social Sisters と同じ見た目」を実現できない（セクション数が多すぎる）。揃える方針との整合が取れない。

第三に、案 C は「LP は刈り込む / 詳細は別ページへ」という Social Sisters の構造を踏襲しつつ、WCC が必要とする情報量（FAQ / service-flow）を別ページで担保できる。

#### Masatoshi 判断項目

- 案 A / B / C のどれを選ぶか
- 案 C 採用時、`/pricing` ページの新設可否（現状 `app/packages/page.tsx` は `/#pricing` リダイレクトのみ。これを実ページに格上げする）
- service-flow と FAQ の `/pricing` への移動を本プランでやるか、別タスクで後追いか

### 論点 5: フッター再設計

**現状**: FooterCtaSection（H2「永遠に残る思い出を捉える」+ CTA）+ SiteFooter（著作権 + 法務リンク）。

**目標**: Social Sisters のフッター構造（左ブランドブロック + 右写真モザイクグリッド）+ フッター CTA セクションのコピー差し替え。

#### 案 A: Social Sisters 完全模倣

**フッター CTA セクション**（フッター直前）:
- H2: 「Capturing memories that will last forever」相当
- 本文: 1 行
- CTA: 「GET IN TOUCH」相当の単一ボタン

→ E（コピー揃え）を採用しない判断のため、**この英語フレーズは使わない**。WCC 独自コピーで置換:

```
H2: 翌朝のふたりへ、ひとつ届ける。
本文: 日程が未定でもご相談いただけます。
CTA: まず相談する
```

**フッター本体**:
- 左半分: ロゴ + ブランド説明文 + USEFUL LINKS（HOME / ABOUT / PACKAGES / GALLERY / CONTACT） + USEFUL BLOGS
- 右半分: 9:16 動画 / 写真モザイクグリッド（PC: 3列 × 2段、SP: 3列 × 2段の正方形）

#### 案 B: フッター CTA セクションのみ更新、フッター本体は据え置き

- フッター CTA: 「永遠に残る思い出を捉える」→ 「翌朝のふたりへ届ける」系に置換
- フッター本体: 現状維持（モザイクグリッドは導入しない）

#### 案 C: 案 A だがモザイクは Phase 2 以降

- フッター CTA セクション: 案 A と同じコピー差し替え + 構造変更
- フッター本体: ブランド説明 + USEFUL LINKS の左半分のみ実装、右半分（モザイク）は **本番動画素材が揃ってから Phase 2 で追加**

#### 私の推奨: **案 C**

理由は 2 つ:

第一に、フッターモザイクグリッドは「Social Sisters が持つ素材の量と質」を伝える装置であり、WCC の現状ダミー素材 6 本ではこの効果が出ない。本番案件動画が 5〜10 件揃ってから実装するのが適切（M2 後半 〜 M3 タイミング）。

第二に、それまではフッター CTA セクション + 左半分のブランドブロックで構造的に Social Sisters に揃えておけば、見た目の一貫性は確保できる。

#### Masatoshi 判断項目

- 案 A / B / C のどれを選ぶか
- 案 C 採用時、フッター CTA H2 の WCC 独自コピー案（私の推奨「翌朝のふたりへ、ひとつ届ける。」/ 別案）
- 法務リンク（特商法 / プライバシー）のフッター内位置（Social Sisters は最下部右、現状 WCC も同じ）
