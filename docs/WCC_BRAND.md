## 1. 屋号

> **For Your Wedding Day**

略称: WCC（内部用のみ、顧客向け不使用）
FYWD 略称: **廃止**（内外の混乱を防ぐため WCC に統一）
ドメイン: `for-your-wedding-day.com`

### 1.1 命名の経緯

既存 LP のドメインと屋号を一致させることで、ブランド・サービス名・URL の三者が一気に揃う設計を採用。新ブランド名選定で時間をかけるより、既に公開され実体のある屋号を確定させ、開発リソースをサービス品質と SNS 実績作りに集中させる判断。

### 1.2 屋号の意味と読み解き

**「あなたの結婚式の日のために」**

- "Your" = 新郎新婦の二人。式は二人のもの、サービス事業者のものでもゲストのものでもない
- "Wedding Day" = その一日。結婚生活全体ではなく、この特別な一日に焦点を絞る
- "For" = 道具的・献身的なニュアンス。私たちはこの一日のために存在する

この三語に、サービスの世界観が凝縮されている。新郎新婦の一日を支えるためにある、という立ち位置。

### 1.3 表記ルール

- 正式表記: **For Your Wedding Day** (各単語の頭文字を大文字、半角スペース区切り)
- 略称表記: **WCC** (内部ドキュメント・データベース内での参照用、顧客に見せない)
- FYWD: **使用禁止**（廃止）
- 日本語表記: 原則使わない。やむを得ず説明が必要な場合は「フォー・ユア・ウェディング・デイ」(中黒区切り)
- ハッシュタグ: `#ForYourWeddingDay` (CamelCase)、`#fywd` (短縮版)
- Eメール署名: `For Your Wedding Day` のみ。「株式会社」等の法人形態は付さない

## 3. ブランドパーソナリティ

For Your Wedding Day が人だったら、こういう人物。

### 3.1 こうある (Are)

- **親密な距離感**: ゲスト席より少し近い、友人より少し丁寧。新郎新婦の半歩後ろから見ている感覚
- **静かな観察者**: 場を仕切らない。式の主役は新郎新婦であり、撮影者は気配を消す
- **手仕事の人**: 機材自慢ではなく、現場の判断と編集の手数で価値を出す
- **誠実な伝え手**: できないことは「できない」と言う。盛らない、嘘をつかない
- **軽やかさ**: 重厚な記録メディアの対極。スマホで撮ってスマホで見る軽さを肯定する

### 3.2 こうではない (Are Not)

- **演出家ではない**: 構図やシーンを組み立てる役割ではない
- **権威ではない**: 「プロのフォト・ビデオグラファー」のような重厚な専門家像を演じない
- **派手な発信者ではない**: トレンドに乗ったコンテンツクリエイターのような派手さは避ける
- **総合ウェディング業者ではない**: 写真・動画・進行・コーディネートの全部はやらない。役割を絞る
- **テック企業ではない**: 「AI 搭載」「プラットフォーム」「DX」のような言葉は使わない

## 4. トーン&ボイス

### 4.1 文章の基本トーン

- **柔らかく、簡潔に**。修飾を重ねず、必要なことを必要なだけ
- **二人称を「おふたり」**または「新郎新婦さま」に統一。「カップル」「お客様」は避ける
- **過剰な敬語を避ける**: 「させていただきます」を連発しない。「いたします」で十分
- **絵文字は最小限**: LINE では場面によっては 🤍 程度を使うが、LP・メールでは原則使わない
- **感嘆符・三点リーダーを乱用しない**: 落ち着いた書き手という印象を保つ

### 4.2 書き分け

| 媒体 | トーン | 例 |
|---|---|---|
| LP | 簡潔・写真主役 | 「翌朝、二人で昨日を笑い合うための動画」 |
| Instagram キャプション | 余白多め・短文連ね | 「式の翌朝。/ コーヒーを淹れて、/ 昨日をもう一度。」 |
| LINE 返信 | 親密だが丁寧 | 「ご相談ありがとうございます。打ち合わせは ZOOM で 30 分ほどお時間いただけますか？」 |
| メール | やや丁寧・要点明示 | 件名は「ご見積もりをお送りします — For Your Wedding Day」 |
| 契約書・特商法表記 | 簡潔な事務文 | 法令準拠、装飾なし |

### 4.3 推奨語と禁句

**推奨する言葉**:

- お二人 / 当日 / 翌朝 / 余韻 / 高まり / 静かに / 軽やかに / 寄り添う / そのまま / 等身大

**避ける言葉** (Tinykomainu の禁句リストに倣う):

- **「プラットフォーム」**: テック企業ぽい、結婚式の文脈に合わない
- **「AI 搭載」「テクノロジー」**: 撮影は人の手仕事
- **「シナジー」「ソリューション」**: 結婚式の人格的な場に合わないビジネス用語
- **「最高の」「最強の」「究極の」**: 誇張表現、信頼を損なう
- **「奇跡の」「感動の」**: 感情を断定する言葉。感じ方は新郎新婦に委ねる
- **「インスタ映え」「バズる」**: 一時的なトレンド語、サービスの落ち着いたトーンに合わない
- **「お客様」「クライアント」**: 距離が遠い。「お二人」を使う
- **「ご利用」「サービス」を多用しない**: 業者然とした距離感を生む。「お手伝い」「ご一緒」等に置き換える

### 4.4 文体の特徴

**文の長さ:**
- ヒーロー見出し: 9〜20 文字、1 メッセージ 1 価値
- リード文: 1 文 25〜50 文字。1 段落は 2 文まで
- FAQ/詳細説明: 1 回答は 3 文以内

**情報の並べ方:** **共感 → 価値 → 条件 → 行動** の順で書く

**数字・価格・時間の書き方:**
- 数字は算用数字で統一し、比較しやすい単位を添える
- 価格は「税込/税別」を明記する
- 納品時間は「いつ受け取れるか」を最優先で示す（例: 当日中・翌日）

### 4.5 見出しの書き方

| 階層 | ルール | 例 |
|---|---|---|
| H1 | サービス名または主要価値を 1 つに絞る。当日の体験が想像できる語を 1 つ入れる | 「翌朝、二人で昨日を笑い合うための動画」 |
| H2 | 「何が分かるか」が即時に伝わる名詞句 | 「料金」「納品までの流れ」「よくある質問」 |
| H3 | 条件や差分を短く示す | 「対応範囲」「当日の動き」「翌日の受け取り」 |

### 4.6 CTA の書き方

CTA は「行動 + 目的」が一目で分かる短文で統一する。

| パターン | 推奨例 |
|---|---|
| 予約導線 | 「予約する」「空き状況を確認する」 |
| 比較導線 | 「プランを比較する」「料金を見る」 |
| 理解導線 | 「サービス内容を見る」「FAQ を見る」 |
| 相談導線 | 「相談する」「問い合わせる」「まず相談する」 |

禁止: 「こちらをクリック」「詳細はこちら」「とりあえず見る」

### 4.7 AI コピーライティング用システムプロンプト

WCC 向け文章生成の基準プロンプト（Claude 等に渡す用）:

```
あなたは For Your Wedding Day の日本語コピーライターです。
目的は、ユーザーが「安心して任せられる」と感じ、比較・理解・予約まで迷わず進める文章を書くことです。

【最優先ルール】
- 温度感は高めで親密に。頼れる伴走者のように話す。
- 感情価値（安心、自然体、取りこぼし防止）を示した後、判断材料（料金・範囲・納品・流れ）を簡潔に示す。
- 1文を短く、1段落2文までに抑える。
- 曖昧語を避け、数字・条件・対象を明記する。
- 誇大表現や煽り表現を使わない。

【トーン】
- 親密で親身。友人のように近いが、言葉は丁寧で誠実。
- 「です・ます」調を基本にする。
- 主語は「おふたり」や「当日の体験」を中心にする。

【文体】
- 共感 → 価値 → 条件 → 行動 の順で書く。
- 見出しは短く、本文は25〜50文字程度を目安にする。

【CTA】
- 行動 + 目的が明確な表現。「予約する」「空き状況を確認する」「料金を見る」「まず相談する」。
- 「こちらをクリック」「詳細はこちら」は使わない。

【禁止】
- 根拠のない最上級（業界 No.1 等）
- 不安を煽る訴求
- 運営都合の長い説明
```

### 5.3 写真・動画のスタイル

**色味**:

- 自然光ベース、暖色寄りのホワイトバランス
- 彩度はやや抑えめ、フィルム調のニュアンス
- 過度な色補正・派手なフィルターは使わない
- 影を黒く潰さず、灰〜茶のグラデーションで残す

**構図**:

- 引きの式場全景より、寄りの表情・手元・足元
- 余白を多く取る (画面いっぱいに被写体を入れない)
- 中央構図より、三分割構図やオフセット配置

**撮影対象の優先順位**:

1. 新郎新婦の二人だけの時間 (誓いのキス、控室、退場後)
2. 親族との交流 (両親への手紙、家族写真の準備中)
3. ゲストの自然な表情 (歓談中、笑い合う瞬間)
4. 式場の雰囲気 (装花、テーブル、光) — ただし主役にしない
5. 式場全景・建物外観 — 必要最小限

## 6. UI/UX 原則

### 6.1 基本原則

- **余白を多く取る。** 「翌朝の二人の時間」のような落ち着きを画面全体で表現する
- **動きは最小限。** ホバーアニメーション・スクロール演出を多用しない
- **写真と動画を主役にする。** 文字情報より実際の作例を前に出す
- **モバイルファースト。** ターゲット層（28〜35 歳の新郎新婦）はスマホから見る前提

### 6.2 コンポーネント仕様（実装値）

**角丸:**

| CSS 変数 | 値 | 用途 |
|---|---|---|
| `--radius-sm` | `2px` | 入力・小ボタン |
| `--radius-md` | `4px` | 輪郭ボタン等 |
| `--radius-full` | `9999px` | Primary CTA |
| `--radius-media` | `2rem` | トップの 9:16 動画カード |
| `--radius-0` | `0` | 例外的な角丸なし |

**ボタン:**

- Primary CTA: 背景 `--color-accent` / 角丸 `--radius-full` / `min-height: 52px` / `min-width: 220px` / `font-size: 14px, font-weight: 600, letter-spacing: 0.08em` / hover: `--color-accent-hover`
- Secondary CTA: `2px solid var(--color-border-strong)` / `min-height: 52px` / hover: 背景 `--color-ink`, 文字 `#fff`
- トップ導線の主 CTA 文言: 「まず相談する」（固定）

**カード:** 影なし・ライン（border）なし・内側余白 `1.5rem`（モバイル `1rem`）

**フォーム:** 入力枠 `2px solid var(--color-border-hairline)` / `min-height: 44px` / focus-visible: `outline: 2px solid var(--color-accent)` + `outline-offset: 2px`

**9:16 メディア:** `aspect-ratio: 9 / 16` / 角丸 `2rem` / ホバーでズーム・ドロップシャドウ禁止

**ヘッダー:** 背景 `--color-canvas` / 高さ `h-20`（SP）/ `md:h-24`（PC）

**フッター:** 背景 `--color-bg-dark` / 見出し・本文 `--color-text-on-dark` / リンク `--color-footer-link`

**深度・エレベーション:**

| レベル | 表現 | 用途 |
|---|---|---|
| 0 | 境界なし | 基本文面 |
| 1 | 1px 境界線 | 区切りのみ |
| 2 | 2px 境界線 | 強調枠/輪郭ボタン |
| 3 | subtle 背景帯 | セクション帯 |
| 4 | accent 色 | CTA/フォーカス/リンク hover |

**禁止:** `box-shadow` / `backdrop-blur` / ガラス風 UI / 可読性目的を超える `text-shadow`

### 7.2 TikTok

- 9:16 縦動画でハイライト 15〜30 秒程度
- 派手な編集・トレンド BGM は使わない
- 「翌朝の余韻」を感じさせる雰囲気重視
- 投稿頻度より品質を優先

### 7.3 共通の禁則

- バズ狙いの過剰な編集はしない
- 流行りの BGM・エフェクトに乗らない
- 「いいね数」「フォロワー数」を SNS 投稿で誇示しない
- ゲスト・新郎新婦の許諾なしに顔が映る素材を投稿しない

## 8. 接客・コミュニケーションの原則

### 8.1 LINE での返信

- 24 時間以内の返信を原則
- 短い返信を即時で返すより、まとまった返信をやや遅めにする方がトーンに合う
- 業務連絡的な要件は箇条書きで整理、感情表現は短く添える

### 8.2 ZOOM 打ち合わせ

- 30 分を基本枠
- 服装はカジュアル寄り (スーツは着ない)、背景は白基調の壁
- 売り込みより聞き取りを主眼に置く
- 当日の進行表・ご家族構成・SNS 利用状況を聞く

### 8.3 撮影当日

- 黒の控えめな服装で式の雰囲気を妨げない
- プランナー・他のフォト/ビデオ業者と挨拶を交わし、進行を妨げない
- ゲストに過度に話しかけない、撮影に集中する
- 食事休憩を新郎新婦が気にしないよう、自分で時間を見て退避する

### 8.4 納品時

- 「お疲れさまでした」より、「素敵な一日でしたね」のような余韻に寄り添う言葉
- 納品 URL とパスワードは LINE で送る
- 納品後 1 週間は閲覧期間、その後の追加要望にも丁寧に対応

## 9. ブランドの一貫性チェックリスト

新しい媒体 (新しい LP セクション、新しい SNS 投稿、新しい契約書フォーマット) を作る時、以下を確認する。

- [ ] メイン体験「翌朝、二人で昨日を笑い合うための動画」と矛盾しないか
- [ ] 「やらないこと」(派手な編集、ドローン、長編ムービー等) を提案していないか
- [ ] 禁句 (プラットフォーム、AI、最高の、奇跡の等) を使っていないか

- [ ] カラーパレットの範囲内か
- [ ] 余白を十分に取っているか
- [ ] 写真・動画が主役で、文字が情報過多になっていないか
